담당 scope: appstore 기능. Backend `api.appstore`, frontend `features/appstore`.

리팩토링 목표:
- 앱 CRUD, screenshot/cover, like, comment/reply 책임을 service/selector 단위로 정리한다.
- base64/data URL/image redirect 처리 로직을 분리하고 validation 중복을 제거한다.
- frontend list/detail/form/filter/carousel 상태와 API mutation hook을 단순화한다.
- 작성자/관리자 권한 check를 읽기 쉬운 helper로 정리한다.

주의:
- 앱/댓글 권한 정책, like toggle semantics, cover endpoint 동작 변경 금지.
- screenshot payload contract 변경 금지.
