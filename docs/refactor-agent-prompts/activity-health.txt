담당 scope: activity + health. Backend `api.activity`, `api.health`, frontend에서 관련 화면이 있으면 해당 feature만.

리팩토링 목표:
- activity log serialization, permission check, limit validation을 간결하게 정리한다.
- middleware에서 생성되는 activity metadata와 조회 API 책임을 분리해 읽기 쉽게 만든다.
- health check는 최소 구현을 유지하고 불필요한 abstraction을 제거한다.

주의:
- ActivityLog 생성 timing, permission requirement, health response contract 변경 금지.
